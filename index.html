What Is US Roulette?

American roulette has 38 pockets: 18 red, 18 black, and 2 green (0 and 00). Each spin is equally likely to land on any pocket.

🤔 Think About It

European roulette has only one green pocket. How would that change the odds?

The Probability Formula

The chance of any event:

P(event) = favorable outcomes ÷ total outcomes P(red) = 18 ÷ 38 ≈ 47.37% P(single number) = 1 ÷ 38 ≈ 2.63%

Red isn't 50% — those two green pockets shift the odds.

House Edge

House Edge = 2 ÷ 38 = 5.26%

For every $100 bet, the casino expects to keep about $5.26 over time.

Expected Value (EV)

EV = (win × P(win)) + (loss × P(lose)) $1 on a single number (pays 35:1): EV = ($35 × 1/38) + (-$1 × 37/38) = -$0.053

Every standard roulette bet has the same negative EV.

Gambler's Fallacy

If red hits 5 times in a row, is black "due"? No! Each spin is independent. The wheel has no memory. P(red) is always 18/38.

Law of Large Numbers

Over many spins, observed frequencies approach theoretical probability. After 10 spins, red might be 70%. After 100,000 spins, it'll be very close to 47.37%. Try it!

Variance

Std Dev = √(n × p × (1-p))

High variance means wild short-term swings even when the long-term average is known. That's why people win sometimes — and always lose long-term.

1. Simulate the Wheel

Model a US roulette wheel in Python.

roulette_sim.py
import random

pockets = [0, "00"] + list(range(1, 37))
red_numbers = {1,3,5,7,9,12,14,16,18,19,21,23,25,27,30,32,34,36}

def spin():
    return random.choice(pockets)

def get_color(result):
    if result in (0, "00"): return "green"
    return "red" if result in red_numbers else "black"

# Spin 20 times
counts = {"red": 0, "black": 0, "green": 0}
for i in range(20):
    result = spin()
    color = get_color(result)
    counts[color] += 1
    print(f"Spin {i+1:>2}: {str(result):>3} -> {color}")
print(f"Results: {counts}")

2. Monte Carlo Convergence

Watch red frequency converge to 47.37% over thousands of spins. Requires matplotlib.

convergence.py
import random, matplotlib.pyplot as plt

pockets = [0, "00"] + list(range(1, 37))
red = {1,3,5,7,9,12,14,16,18,19,21,23,25,27,30,32,34,36}
N = 10_000
rc, freq = 0, []
for i in range(1, N+1):
    if random.choice(pockets) in red: rc += 1
    freq.append(rc / i)
plt.plot(freq, color='crimson')
plt.axhline(y=18/38, color='gold', linestyle='--')
plt.xlabel('Spins'); plt.ylabel('Red Frequency')
plt.title('Law of Large Numbers'); plt.show()

Each face is equally likely:

P(any face) = 1/6 ≈ 16.67%

Two Dice — Sum Probabilities

With 2 dice: 36 combinations, but sums aren't equally likely!

7 is the most common sum — 6 ways to make it!

Expected Value

E[one die] = (1+2+3+4+5+6) ÷ 6 = 3.5 E[two dice] = 3.5 + 3.5 = 7.0

Complementary Probability

P(at least one 6 with 2 dice): P(no sixes) = 5/6 × 5/6 = 25/36 P(at least one 6) = 1 - 25/36 = 11/36 ≈ 30.6%

The Simplest Experiment

P(Heads) = 1/2 = 50% P(Tails) = 1/2 = 50%

A coin flip is a Bernoulli trial — exactly two outcomes.

Binomial Distribution

P(k heads in n flips) = C(n,k) × p^k × (1-p)^(n-k) P(3 heads in 5 flips) = C(5,3) × 0.5³ × 0.5² = 10 × 0.125 × 0.25 = 31.25%
🤔 Think About It

What's the probability of 5 heads in a row? (0.5&sup5; = ?)

Multiplication Rule

P(A and B) = P(A) × P(B) (independent events) P(3 heads in a row) = 1/2 × 1/2 × 1/2 = 1/8 = 12.5%

Streaks Are Normal

In 100 flips, expect a streak of ~7. Humans think this looks "not random" but it's perfectly normal. Fake random sequences lack long streaks.

Flip Coins & Analyze

coins.py
import random
from math import comb

# Flip 1000 coins
results = [random.choice(["H","T"]) for _ in range(1000)]
heads = results.count("H")
print(f"Heads: {heads}/1000 = {heads/10:.1f}%")

# Binomial: P(exactly k heads in n flips)
n, p = 10, 0.5
for k in range(n+1):
    prob = comb(n,k) * p**k * (1-p)**(n-k)
    print(f"P({k:2} heads) = {prob*100:5.2f}%")

52-Card Deck

4 suits × 13 ranks = 52 cards P(any card) = 1/52 ≈ 1.92% P(heart) = 13/52 = 25% P(face card) = 12/52 ≈ 23.1%

Dependent Events

P(1st Ace) = 4/52 P(2nd Ace | 1st was Ace) = 3/51 P(both Aces) = 4/52 × 3/51 = 0.45%

Unlike dice/coins, drawing cards changes future probabilities.

🤔 Think About It

After drawing 5 hearts, are you more or less likely to draw another? Why isn't this the Gambler's Fallacy?

Combinations

C(n,k) = n! ÷ (k! × (n-k)!) Poker hands: C(52,5) = 2,598,960

Simulate Card Drawing

cards.py
import random
ranks = ['A']+[str(i) for i in range(2,11)]+['J','Q','K']
suits = ['♥','♦','♠','♣']
deck = [(r,s) for s in suits for r in ranks]
random.shuffle(deck)

hand = [deck.pop() for _ in range(5)]
print("Hand:", [f"{r}{s}" for r,s in hand])

aces = sum(1 for r,s in deck if r=='A')
print(f"P(next Ace) = {aces}/{len(deck)} = {aces/len(deck)*100:.2f}%")

Classic Probability

Bag: 5 red, 3 blue, 2 green = 10 total P(red) = 5/10 = 50% P(blue) = 3/10 = 30% P(green) = 2/10 = 20%

With vs. Without Replacement

With replacement: P stays the same (independent) Without replacement: P changes (dependent) Drawing 2 reds from 5R+5B: With: 5/10 × 5/10 = 25.0% Without: 5/10 × 4/9 = 22.2%

Conditional Probability

P(A | B) = "P of A given B happened" P(2nd red | 1st was red) = 4/9 P(2nd red | 1st was blue) = 5/9

Marble Simulation

marbles.py
import random
from collections import Counter

bag = ["red"]*5 + ["blue"]*3 + ["green"]*2

# With replacement
draws = [random.choice(bag) for _ in range(10_000)]
print("With replacement:")
for c, n in Counter(draws).most_common():
    print(f"  {c}: {n/100:.1f}%")

# Without replacement: P(2 reds in a row)
N = 100_000
hits = sum(1 for _ in range(N)
    if random.sample(bag,2) == ["red","red"])
print(f"P(2 reds w/o replacement): {hits/N*100:.2f}%")
print(f"Exact: {5/10*4/9*100:.2f}%")
